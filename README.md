# Convoy of Hope GitHub Pages Version

This version is prepared for GitHub Pages. It uses Supabase instead of PHP.

## 1. Create Supabase table
Run this SQL in Supabase:

```sql
create table public.feedback (
  id bigint generated by default as identity primary key,
  category text not null,
  rating int not null check (rating between 1 and 5),
  message text not null,
  created_at timestamptz not null default now()
);
```

## 2. Add policies
Enable Row Level Security on the table, then run:

```sql
alter table public.feedback enable row level security;

create policy "Allow anonymous inserts"
on public.feedback
for insert
to anon
with check (true);

create policy "Allow anonymous reads"
on public.feedback
for select
to anon
using (true);
```

## 3. Add your Supabase keys
Edit `config.js` and fill in:
- `supabaseUrl`
- `supabaseAnonKey`

## 4. Upload to GitHub
Create a GitHub repo and upload these files:
- `index.html`
- `app.js`
- `config.js`
- `styles.css`
- `manifest.json`
- `sw.js`
- `icons/`

## 5. Enable GitHub Pages
In the GitHub repo:
- open `Settings`
- open `Pages`
- Source: `Deploy from a branch`
- Branch: `main`
- Folder: `/ (root)`

GitHub will give you a public HTTPS URL.

## Important
`config.js` contains your public anon key. That is expected for a browser app, but only use the Supabase anon key, never the service role key.
